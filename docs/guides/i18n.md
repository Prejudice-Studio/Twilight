# 前端多语言开发与翻译指南

本文说明 Twilight WebUI 的多语言资源组织、locale 命名、开发接入与翻译维护流程。

## 资源目录

前端语言文件统一存放在：

```text
webui/src/locales/
```

当前使用 JSON 作为语言资源格式，原因是 Next.js / TypeScript 可直接 import，并能在编译期校验 key 结构。现有文件：

| 文件 | locale | 说明 |
| ---- | ------ | ---- |
| `basic.json` | 不对用户展示 | 底层基准结构与 fallback 文案 |
| `zh-Hans.json` | `zh-Hans` | 简体中文，默认显示语言 |
| `zh-Hant.json` | `zh-Hant` | 繁體中文 |
| `en-US.json` | `en-US` | English (United States) |

`basic.json` 是唯一的 key 结构基准。不要使用任一具体显示语言（包括 `zh-Hans`）作为类型基准或 fallback 基准。
主分支只提供并维护 `zh-Hans`、`zh-Hant`、`en-US` 三个显示语言，其他语言不在主分支内置。

## Locale 命名规范

- 使用 BCP 47 风格命名，例如 `zh-Hans`、`zh-Hant`、`en-US`。
- 文件名必须与 locale 名称一致：`<locale>.json`。
- 不使用下划线命名，例如不要写 `en_US.json`。
- 中文优先使用脚本子标签区分简繁：`zh-Hans` / `zh-Hant`，不要新增 `zh-CN` 作为主语言文件。

浏览器语言和历史本地缓存会在 `webui/src/lib/i18n.tsx` 中归一化，例如 `zh-CN` 会映射到 `zh-Hans`，`zh-TW` 会映射到 `zh-Hant`。

## 接入方式

根布局已经挂载 `LocaleProvider`。页面或组件中读取文案：

```tsx
import { useI18n } from "@/lib/i18n";

export function Example() {
  const { t } = useI18n();
  return <span>{t("settings.localeTitle")}</span>;
}
```

带参数文案使用 `{name}` 占位：

```json
{
  "signinRenewal": {
    "actionLabel": "{cost} {currencyName} 续 {days} 天"
  }
}
```

```tsx
t("signinRenewal.actionLabel", {
  cost: 100,
  currencyName: "积分",
  days: 30,
});
```

## 新增或修改文案

1. 先在 `webui/src/locales/basic.json` 添加或调整 key，作为底层 fallback 和 TypeScript 结构基准。
2. 同步在 `zh-Hans.json`、`zh-Hant.json`、`en-US.json` 中添加或调整对应翻译。
3. 显示语言文件允许只提供 partial overlay；缺失 key 会安全回退到 `basic.json`。
4. 在组件中使用 `t("section.key")`，不要在业务代码中硬编码新增的可见文案。
5. 执行 `cd webui && pnpm lint && pnpm build`，确认 key 结构和类型检查通过。

TypeScript 使用 `basic.json` 作为基准结构；显示语言缺 key 时回退到 `basic.json`，结构类型不兼容时构建会失败。

## 新增语言

主分支不内置或维护其他显示语言。需要更多语言时，应在自维护分支中添加对应 locale 文件并调整 `webui/src/lib/i18n.tsx`，不要把非简繁英语言资源合入主分支。

## 翻译约定

- key 使用稳定业务语义，不要把展示文案本身作为 key。
- key 层级保持简短，例如 `common.githubProject`、`settings.localeTitle`。
- 表述应尽量避免政治或归属判断歧义。语言名称只描述语言、文字或脚本，例如使用 `zh-Hans` / `zh-Hant`，不要把语言选项写成归属标签。
- 不要根据用户输入动态拼接 key；用户输入只能作为 `t()` 的参数值。
- 翻译文件只放纯文本，不放 HTML、React 组件或 CSS。
- 涉及 Token、密码、API Key、服务器线路等敏感信息时，只翻译提示文案，不在语言文件中写真实值。
- 需要复数、日期和数字格式时，优先在组件里用 `Intl.*` 按当前 locale 格式化，再传入 `t()`。

## 语言切换入口

- 通用切换组件：`webui/src/components/locale-switcher.tsx`。
- 当前入口位于桌面侧边栏底部和移动端菜单底部，紧邻浅色 / 暗色主题切换按钮。
- 用户偏好保存在当前浏览器的 `localStorage` key：`twilight:locale`，不依赖后端会话。
